# PeakPrep

AI-powered SAT prep with a mountain-climbing theme. PeakPrep helps students practice, plan, and track progress toward their target score.

**Live app:** [study-app-nu-two.vercel.app](https://study-app-nu-two.vercel.app)

---

## What the app does

PeakPrep has four core tools, all reachable from the sidebar after sign-in:

| Tool | Route | Purpose |
|------|-------|---------|
| **Dashboard** | `/dashboard` | Home base — streak, weekly goal, today’s study-plan tasks, score snapshot |
| **Practice Tests** | `/practice` | AI-generated Math or Reading & Writing checkpoints |
| **Study Plans** | `/study-plan` | Personalized multi-week expedition plans |
| **Micro-Lessons** | `/lessons` | Short skill lessons with embedded practice |
| **Progress** | `/progress` | Score history, trends, skill domains, target goal |

The public landing page (`/`) explains the product. Sign-in is required for AI generation.

## How each feature works

### Practice Tests

1. Pick **Math** or **Reading & Writing**, then configure question count (5–25), optional topic, and difficulty.
2. The API generates SAT-style multiple-choice questions via OpenAI (`gpt-4o-mini`).
3. **Progressive loading:** tests with more than 5 questions start with 2 questions, then load the rest in batches of up to 5 in the background using the same `existingTestId`. A status banner shows `(X of Y ready)` while loading; if batches fail after retries, you can **Continue anyway** or **Retry loading**.
4. **In-test UX:** per-question confidence (Got it / Unsure / No idea), optional timer, Desmos graphs for math, split passage layout for R&W, built-in calculator panel for math.
5. **Trail Buddy:** ask follow-up questions about the current item during review.
6. **Session recovery:** in-progress tests are saved to `localStorage`. On refresh, a banner offers **Resume** or **Start fresh** — nothing auto-resumes silently.
7. **Results:** raw score (X/Y), skill breakdown rings, **Your Weak Spots** (top miss-rate skills from history), honest scaled **ranges** (not fake point estimates on short tests), score trend vs prior session, streak/weekly-goal nudges, and deep links back to practice or lessons.
8. **Quality controls:** question fingerprints prevent repeats across sessions; topic/difficulty locks; calibrated difficulty wording; R&W passage domain rotation.

### Study Plans

1. Enter target score, test date, weekly hours, and focus areas.
2. AI generates a **weekly** plan (up to 6 weeks server-side); daily tasks are derived locally — not generated by the LLM on every request.
3. Tasks appear on the dashboard with completion pills stored in `localStorage`.
4. Plans can be regenerated; completion state is tracked per task ID.

### Micro-Lessons

1. Choose a topic (aligned to SAT skill domains).
2. AI generates a short lesson: goal, explanation bullets, example, and practice questions.
3. Completing a lesson marks it done locally and surfaces a CTA to practice that skill.

### Progress & Dashboard

- **Scores** sync to the database when signed in; anonymous fallback uses `localStorage`.
- **Estimated SAT score** and trends require enough completed questions (see `MIN_ESTIMATE_QUESTIONS` in `src/utils/scoreTracking.ts`).
- **Streak** and **weekly goal** are client-side retention hooks (`src/utils/streakTracking.ts`, `src/utils/weeklyGoal.ts`).
- **Skill weaknesses** aggregate per-skill attempts/misses on the user record after each scored practice test.

## Accounts, free tier, and Plus

- **Authentication:** email + password (bcrypt), session cookies via Prisma `Session` model.
- **Sign-in required** for all AI endpoints (practice, study plan, lessons).
- **Free tier:** **1 AI generation per calendar month** across practice tests, study plans, and micro-lessons combined. Enforced server-side in `src/utils/premiumGate.ts` by counting DB records — not client `localStorage`.
- **PeakPrep Plus (~$5/mo):** unlimited AI generation via Stripe subscription (`ACTIVE` or `TRIALING` status).
- **Gate UX:** pages fetch `GET /api/user/access-status`; hitting the limit returns HTTP 402 and opens `PremiumGateModal`.

Dev-only bypass: set `BYPASS_PREMIUM_GATE=true` when `NODE_ENV !== "production"`.

## Getting started (local)

### Prerequisites

- Node.js 20+
- OpenAI API key (required for AI features)
- Stripe keys (optional — app boots without them; payments disabled)

### Setup

```bash
git clone <your-repo-url>
cd study-app
npm install
```

Create a `.env` file in the project root (not committed):

```env
DATABASE_URL="file:./dev.db"
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# Required for AI features
OPENAI_API_KEY=sk-...

# Required for payments (optional locally)
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
STRIPE_PRICE_ID_MONTHLY=price_...
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...

# Optional
NEXT_PUBLIC_DESMOS_API_KEY=...
BYPASS_PREMIUM_GATE=true
```

Initialize the database:

```bash
npx prisma migrate dev
```

Run the dev server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

`npm install` runs `prisma generate` automatically via `postinstall`.

## API overview (main routes)

| Method | Route | Notes |
|--------|-------|-------|
| POST | `/api/generate-practice` | New test or batch append via `existingTestId` |
| POST | `/api/generate-personalized-plan` | Weekly study plan |
| POST | `/api/ai/lessons` | Micro-lesson content |
| PATCH | `/api/practice-tests/[id]/score` | Save score + update `skillWeaknesses` |
| GET | `/api/user/access-status` | `{ allowed, isPremium }` |
| GET | `/api/user/skill-weaknesses` | Aggregated weak skills |
| POST | `/api/stripe/create-checkout` | Start Plus subscription |

Mutating API routes are CSRF-protected via origin check in `src/middleware.ts`.

## Deploying to Vercel (Turso)

1. Create a Turso database at [turso.tech](https://turso.tech).
2. Set Vercel environment variables:
   - `DATABASE_URL` — Turso libSQL URL
   - `TURSO_AUTH_TOKEN` — Turso auth token
   - `OPENAI_API_KEY`, Stripe keys, `NEXT_PUBLIC_APP_URL`, etc.
3. Apply migrations to Turso (Prisma CLI does not accept `libsql://` for `migrate deploy`):

```powershell
$env:DATABASE_URL="libsql://your-db.turso.io"
$env:TURSO_AUTH_TOKEN="your-token"
node scripts/apply-turso-migrations.mjs
```

4. Deploy:

```bash
npx vercel --prod
```

Or push to the connected Git branch if CI deploy is configured.

### Stripe webhooks (production)

1. Stripe Dashboard → Developers → Webhooks
2. Endpoint: `https://your-domain.com/api/webhooks/stripe`
3. Events: `checkout.session.completed`, `customer.subscription.*`, `invoice.payment_*`
4. Copy signing secret → `STRIPE_WEBHOOK_SECRET`

### Local Stripe webhooks

```bash
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

## Security notes

- Security headers in `next.config.ts` (X-Frame-Options, etc.)
- Auth routes rate-limited in-memory (`src/lib/rate-limit.ts`) — resets on serverless cold starts
- Stripe initializes to `null` without keys; payment UI degrades gracefully
- OpenAI client is lazy-loaded so builds succeed without `OPENAI_API_KEY`
